Spectre AI

An AI-powered API mock proxy that learns from real traffic

In Development — Oahu, Hawaii

what is it?

Spectre AI is a drop-in reverse proxy for developer and QA environments. Point it at any backend service and it automatically records real traffic, then replays those interactions as accurate, deterministic mocks — no manual fixture writing required.

When a request doesn't match any recorded mock, a fine-tuned AI model steps in and generates a realistic response on the fly, so tests never fail on missing coverage.

three modes, one proxy

Learn

Learn Mode

All traffic is proxied to the real backend and every request/response pair is recorded to a persistent mock library. Deduplicates by method + URI automatically.

Mock

Mock Mode

Serves recorded responses instantly from the in-memory library. Falls back to response sequences for stateful flows, then to AI generation if no match is found.

Passthrough

Passthrough Mode

Zero interception — traffic flows straight to the upstream. Useful for prod-like environments where you want routing but no mocking side-effects.

tech stack

Rust Agent

High-performance reverse proxy built on Axum + Tokio. Handles routing, mock matching, and AI fallback with a circuit breaker.

AI Inference

Fine-tuned language model (Qwen-based) served via a Python inference server. Generates realistic API responses with confidence scoring.

Control Plane

FastAPI backend managing services, patterns, traffic snapshots, training triggers, and multi-tenant access.

Dashboard

Next.js + Tailwind UI for browsing mocks, inspecting traffic, managing sequences, and triggering model retraining.

architecture

Proxy Agent

Stateless Rust service at the edge. Matches incoming requests against an in-memory mock library, handles response sequences, and calls the inference server when needed.

Rust Axum Tokio Prometheus
AI Inference Server

Distilled fine-tune trained on recorded traffic. Returns a structured response with a confidence score. A circuit breaker in the agent prevents cascading failures if the model is unavailable.

Python PyTorch Transformers Docker
Control Plane API

Manages the mock library, service groups, pattern definitions, and training data. Syncs the agent's JSONL files to a relational database for querying and analytics.

FastAPI SQLAlchemy PostgreSQL
Dashboard

Real-time view into mock coverage, unmatched traffic, and model training status. Lets engineers add mocks manually, define response sequences, and push new training runs.

key features

  • Auto-records real API traffic with header redaction for sensitive fields
  • Fuzzy path matching — handles UUIDs, IDs, and query param variations
  • Response sequences for stateful multi-step flows (e.g. order lifecycle)
  • Confidence threshold configurable at runtime — tune AI fallback aggressiveness
  • Multi-stream routing — route by host or path prefix to different upstream services
  • Prometheus metrics + /health endpoint out of the box
  • Fully containerized — runs locally or in CI with a single docker-compose up
  • Mode persists across restarts — no re-configuration after deploys
